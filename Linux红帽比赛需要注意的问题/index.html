发表于 2019-09-21 更新于 2019-10-06 分类于 Linux ， Linux需要注意的问题 评论数： 阅读次数：

查看本机的语言 echo $LANG，locale （查看本机安装的语言包）也可以设置ssh工具的语言编码

第0道题

  1. 需要注意touch /.autorelabel的拼写
  2. 修改网卡的网络连接的配置信息，如果网卡的名字是system eth0,那么nmcli con modify时 要加双引号”
  3. connection的配置信息的文件为：/etc/sysconfig/network-scripts/ifcfg-ens33 （路径最后一个为网卡的名）
  4. 注意：修改配置文件时，添加ip为IPADDR1= 添加掩码为：PREFIX1= （需要注意的是IPADDR的1表示一个connection可以有多个Ip，那么PREFIX1后面的数字要对应），网关：GATEWAY1= DNS1=
  5. 记得重启网络 systemctl restart network

第一道题：设置selinux

  1. 永久生效 vim /etc/selinux/config 设置为enforcing

第二道题：yum源 /etc/yum.repo/*.repo

  1. yum-config-manager –add <加源的地址>
  2. 然后在往新生成的.repo文件中增加gpgcheck=0，执行yum clean all ,查看包数量–yum repolist

第三道题：扩容LVM

查看命令：lsblk df -Th pvs 、vgs lvs

  1. 首先查看vg是否有剩余空间，如果有，直接执行lvextend -L 700M /dev/vg1/lvm1 （这个路径可以查看）

  2. 创建好分区后，使用pvcreate /dev/sdb2，如果未找到设备，可以使用partprobe

  3. 可以使用fdisk -l /dev/sdb查看磁盘的信息，也可以进入fdisk /dev/sdb进入操作界面，p打印信息查看，

  4. fdisk -l查看所有磁盘的信息

  5. 问题：需要将分区的类型从默认的linux改为8e linux lvm 就能创建lvm吗

    答案是不能：如果不修改分区的系统类型，重启就会进入紧急模式，可能是因为不能正确挂载然后出现错误

  6. 注意：lv + -L指定大小，-n指定名字

    lvcreate -L 500M vg2 -n lvm2 lvextend -L 600M /dev/vg2/lvm2

    重新识别文件大小resizeof

第四道题：用户与组

  1. echo 123 | passwd –stdin user1

  2. 跟用户相关的三个文件

    /etc/passwd /etc/group /etc/shadow

第五道题：文件权限

  1. 修改文件的所有者和所有组 chown : <文件>

    ACL特殊权限

第六道题：周期任务（分时日月周）

crontab -e -u natasha -e表示编辑，u表示用户

第七到题：目录权限

  1. 记住UGOA
  2. 注意特殊权限 chomd g+s <目录> 该目录下的文件自动继承group名

第八道题：更新内核

  1. 注意wget命令，下载文件是下载到当前目录，如果没有指定-O，那么这个文件的默认命名就是网络链接的最后一个 / 后面的字符
  2. 然后安装，rpm -ivh <包名>
  3. uname -r 查看
  4. rpm -ivh –force表示强制安装
