python网络爬虫-scrapy框架学习

发表于 2020-01-20 更新于 2020-01-25 分类于 python网络爬虫 评论数： 阅读次数：
本文字数： 3.5k 阅读时长 ≈ 3 分钟

组件

  • 引擎（scrapy engine）
  • 调度器（scheduler）
  • 下载器（download）：获取网页
  • 爬虫器（spider）：解析网页
  • 管道（item pipeline）：存储数据
  • 下载器中间件（download middlewares）
  • 爬虫中间件（spider middlewares）

数据流

数据流的流动方向

  1. 引擎：向爬虫器请求第一个要抓取的网页url
  2. 爬虫器：提供url给引擎
  3. 引擎：接受到网址，交给调度器排序入队
  4. 调度器：将它处理成请求request给引擎
  5. 引擎：接受到request，并通过下载器中间件给下载器下载（也就是解析网页）
  6. 下载器：根据request下载页面，返回对应的回应（response）给引擎
  7. 引擎：接受到response，并通过爬虫器中间件给爬虫器处理
  8. 爬虫器：处理response，提取数据，返回结果item给引擎，如果有跟进的请求request也会给引擎
  9. 引擎：接受到item，交给管道；新的request给调度器
  10. 管道：存储数据

如果有新的request给调度器，那么从第2步开始重复直到调度器没有request，引擎就会关闭本次爬虫。
