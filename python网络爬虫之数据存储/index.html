将数据存放在mysql数据库

  • 创建数据库create database scraping;

  • 切换数据库use scraping;

  • 创建数据表create table 表名(column_name1 column_type1, column_name2 column_type2);

  • 查看表的结构describe 表名;

  • 在数据表中插入数据insert into urls (url, content) values ("www.baidu.com", "这是内容");

  • 查看数据表

    • select * from urls;
    • select url, content from urls where id =1;
    • select * from urls where id=1

  • 删除数据delete from urls where ulr="www.baidu.com";如果不加where子句，就会将整个数据表全部删除

  • 插入数据insert into urls (url,content) values ("www.baidu.com", "陈晓健");因为之前创建的第一条数据被删除了，然后现在重新创建了新的一条，就默认从第二开始算起。

  • 修改数据update urls set url="www.google.com",content="google" where id =2;

MongoDB与SQL的区别和联系
SQL术语 MongoDB术语 解释
database database 数据库
table collection 数据库表/集合
row document 数据记录行/文档
column field 数据字段/域
index index 索引
table_joins MongoDB不支持表的连接
primary_key primary_key 主键，MongoDB自动将_id字段设置为主键

MongoDB的文档不需要设置相同的字段，并且相同的字段不需要相同的数据类型

使用python操作MongoDB
  • 首先连接客户端
  • 然后连接数据库
  • 最后连接对应的集合（也就是表）
  • pip install pymongo安装操作库

  • 创建数据库 数据库只有在内容插入后才会创建，也就是说数据库创建后还要创建一个集合并插入一个文档，数据库才可以创建，也就是说，如果你现在去判断该数据库是否存在，答案是否定的

  • 判断数据库是否存在
  • 创建集合

集合只有在内容插入后才会创建! 就是说，创建集合(数据表)后要再插入一个文档(记录)，集合才会真正创建。

  • 判断集合是否存在

  • 插入文档

    • insert_one(dict)：插入一条文档，参数是字典对，返回一个InsertOneResult，该对象有inserted_id属性，表示插入文档的id值

    • insert_many(字典列表)：插入多个文档，参数是字典列表，返回一个InsertManyResult对象，该对象有inserted_ids属性

    • 指定_id插入多个文档

  • 查询文档

    • 查询一条数据find_one()，返回集合中第一条数据
    • 查询集合中所有数据find()
    • 查询指定字段的数据，将要返回的字段对应值设置为1，除了_id字段外，其他字段同时指定为0， 1则会报错
    • 根据指定条件查询
    • 修饰符查询大于修饰符$gt
    • 使用正则表达式查询，只用于搜索字符串的字段，$regex
    • 返回指定条数记录， limit()方法

  • 修改文档

    • update_one()方法：第一个参数为查询的条件，第二个参数为要修改的字段，如果查找到的匹配数据多于一条，则会修改第一条，
    • 参数upsert=True表示当没有查询到符合的数据，就新数据插入
    • update_many()：修改所有匹配到的记录

  • 排序sort方法，第一个字段指定要排序的字段，第二个字段指定排序的规则，1位升序，-1为降序，默认为升序

  • 删除数据

    • 删除单个文档delete_one
    • 删除多个文档delete_many()
    • 删除所有的文档当传入的参数是空对象，就会默认将集合中的文档全部删除{}
    • 删除集合drop()方法, 删除成功返回True，否则返回false

    查看是否删除成功show tables
